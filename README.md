/ - перемещаться через несколько директорий

touch - создать файл
    Если использовать символы ~ : touch ~/ss.txt - файл создастся в родительской директории
                             .. : touch ../ss.txt - файл создастся на уровень выше и так по накатаной

mkdir - создать директорию
    -p - создает несколько директорий (вложенность)
    Если использовать символы ~ : mkdir ~/con - создаст папку в родительской директории
                             .. : mkdir ../con2 - создаст папку в предыдущей директории

$git init - инициализировать папку как репозиторий
            $ rm -rf .git - разгитить папку, то есть удаляем сктытую подпапку .git
            ключ -f (от англ. force — «заставить») избавит вас от вопросов вроде «Вы точно хотите удалить этот файл?
                                                   А этот? И этот тоже?».
            .git - хранится история изменений

$git add - подготовка к сохранению (только запоминает текущее содерижимое)
        --all - сразу все файлы
        txt.txt - один файл
        . - всю текущую папку

$git status - проверили статус (состояния)/untracked)) - значит, что Git ещё не хранит информацию о версиях файла
                                                         и не может отследить, как он изменялся.
коммитом - называется сохранение или фиксация состояния файла.  «Сделать коммит» значит сохранить текущую версию файла.

modified - если отредактировать файл (после команды $git add) - он будет как и зелёной, так и в красной зоне

$ git commit -m 'Мой первый коммит!' - c ключом -m (от англ. message — «сообщение»), который присваивает коммиту
                                       сообщение.

      giasu@Worker:~/dev/first-project$ git commit -m 'Мой первый коммит!'
      [main (корневой коммит) 05681cd] Мой первый коммит! - root-commit — это самый первый, или «корневой» (англ. root),
                                                                          коммит в ветке, у следующих коммитов такой надписи не будет;
                                                                          5681cd - сокращённый идентификатор коммита.
       2 files changed, 1 insertion(+) - изменились два файла (readme.txt и todo.txt);
       Create mode 100644 readme.txt - mode 100644 - сообщает, что это обычный файл.
       create mode 100644 todo.txt

      Если выполнить команду git commit без параметра -m, вы попадёте в другую программу — редактор текста vi. Таким образом
      git предлагает вам ввести сообщение о коммите:
        vi - может работать в двух режимах: управление и вставка. В режиме управления vi запускается по умолчанию.
        В этом режиме вы не сможете ввести текст, но в нём работают команды управления, вызываемые с помощью символа:
        Самое простое, что вы можете сделать, попав в vi при коммите, это выйти из него с помощью команды :q!,
        где «q» означает Quit (англ. уйти), а восклицательный знак «!» подтверждает, что нужно выйти без сохранения
        изменений — не важно, были ли эти изменения сделаны или нет. Если же вы всё же решитесь освоить vi,
        важно запомнить, что переход в режим вставки выполняется с помощью кнопки i (от англ. Insert — вставить).
        В этом режиме вы можете набрать нужное вам сообщение. Затем нужно с помощью кнопки Esc перейти в режим
        управления и выполнить команду :wq — она сохранит изменения и закроет редактор.


$git log - посмотреть историю коммитов.

SSH - сетевой протокол дял безопасного обмена данными в сети. Использует пару ключей:
            Приватный - хранится на моём компе (используется для расшифровки данных)
            Публичный ключ - доступен всем и используется для шифрования данных

$ git clone - копирует репозиторий (удалённый) - автоматически связывает локальный и удалённый репозитории

$ ls -la .ssh - проверка на наличие ключа

$ ssh-keygen -t ed25519 -C"email.github" - используется программа ssh-keygen для генерации SSH-пары
            ed25519 - алгоритм шифрования

$ ls -a ~/.ssh - проверка на генерацию ключей
            .pub - публичный ключ

$git remote add origin git@github.com:ИМЯ_АККАУНТЫ/папка с локальным репозиторием
                                            - привязка удалённого репозитория с локальным.
            origin - стандартный псевдоним, с помощью которого можно обращаться к главному удалённому репозиторию

$git remote -v - убедится что репозитории связаны
            -v - короткая форма флага --verbose. Позволяет показать больше информации в выводе

$git push (-u) origin... - позволяет загрузить содержимое локального репозитория на github.
                        ...main - имя первой ветки, её имя нужно указывать при отправке коммитов на удаленный
                                  репозиторий или при получении их из него
                        -u - нужно указывать флаг, для связи локальной ветки с одноимённой удалённой

таблицы:
            Вертикальные линии обозначают столбцы.

            | Таблицы       | Это                | Круто |
            | ------------- |:------------------:| -----:|
            | столбец 3     | выровнен вправо    | $1600 |
            | столбец 2     | выровнен по центру |   $12 |
            | зебра-строки  | прикольные         |    $1 |

            Внешние вертикальные линии (|) не обязательны, и они нужны только чтобы сам код Markdown
            выглядел красиво. Тот же код можно записать так:

            Markdown | не такой | красивый
            --- | --- | ---
            *Но выводится* | `так же` | **клево**
            1 | 2 | 3
